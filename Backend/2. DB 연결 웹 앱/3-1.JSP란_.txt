jsp는 <%와 같은 약속된 몇 가지 기호들을 가지고 있고,
그 기호가 뭐냐에 따라 jsp가 서블릿으로 바뀔 때
어떻게 바꿔야 할지 결정됨
왜냐하면 모든 jsp는 그 자체로 동작하는 게 아니라 서블릿으로 바뀌어서 동작하기 때문

ex)
<%@ : 지시자
<%@ page : page 지시자
-> <%@ page ...... %> : page 지시문
language : 이론적으로 jsp는 내부적으로 자바 외에 다른 언어들도 사용 가능.
           그러나 jsp에서 사용되는 언어는 자바 뿐
contentType : 서블릿의 response.setContentType과 같은 역할
charset : 브라우저에게 지금 출력되는 결과가 어떤 문자셋으로 되어있는지 알려줌
pageEncoding : 이 jsp파일 자체가 어떤 문자셋으로 되어 있는지

이런 지시를 듣는 것은 JSP를 실행하는 WAS

지시문 다음에는 보통 html코드들이 나오게 됨
jsp를 쓰는 이유는, html 안에서 프로그램을 실행시켜주고 싶기 때문
그렇기에 jsp에서는 html 안에 java코드를 넣을 수 있음

<% %> : Scriptlet. java코드를 입력할 수 있는 부분

그러나 서블릿에서 java코드를 실행시킨다고 해도
response의 out.println에 넣어주지 않으면 실제 응답결과로는 들어가지 않음

<%= %> : 표현식. 응답 결과에 넣고 싶은 java코드를 입력하는 부분. out.println()과 같다

jsp를 볼 때는 이게 서블릿으로 바뀔 때 어떻게 바뀔까?를 생각하면 됨


[JSP등장 배경]

- 마이크로소프트에서 ASP(Active Server Page)라는,
  쉽게 웹을 개발할 수 있는 스크립트 엔진을 발표(1998년)
- 1997년에 발표된 서블릿은 ASP에 비해 상대적으로 개발방식이 불편
- ASP에 대항하기 위해 1999년 썬마이크로시스템즈에서 JSP를 발표
- JSP는 실제로 서블릿 기술을 사용
